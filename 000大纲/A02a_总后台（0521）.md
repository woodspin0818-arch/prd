# 小程序SaaS营销平台 总后台（平台运营端）：功能清单与页面框架

> **设计说明**：本文档为小程序SaaS营销平台「总后台」（平台运营端 Admin Web）的功能清单与页面框架规划。总后台由 SaaS 平台运营人员使用，负责管理所有商户客户、配置营销活动、管控小程序部署流程、配置全局权益规则、查看平台运营数据。
>
> **架构定位**：`总后台（平台运营端） → 客户管理后台（商户自助端） → C端小程序`
>
> 本文档仅覆盖「总后台」。商户自助端的功能清单与页面框架见《A02b_客户管理后台全局说明》。

## 一、功能清单

| 功能ID | 功能名称 | 功能描述 | 对应G模块 | 优先级 | 关键业务规则 | 时间 |

### 2.3 核心模块概要说明

---

#### B01 数据看板

**核心指标卡片**：

| 指标 | 说明 |
|------|------|
| 累计注册客户数 | 所有已注册商户总数 |
| 部署完成率 | 已上线数 / 总注册数 × 100% |
| 活跃活动数 | 当前进行中的活动数量 |
| 今日推送点击率 | 今日推送点击数 / 推送成功数 × 100% |

**趋势图区**：近30天注册客户趋势、近30天取链次数趋势、各活动取链排行。

---

#### B02 客户管理

**列表页字段**：客户ID、公司名称、注册手机号（脱敏）、小程序数量、部署状态（待部署/部署中/已上线）、推广位数量、注册时间、状态（正常/停用）、操作（查看详情）。

**默认排序**：注册时间倒序。

**筛选条件**：手机号搜索、公司名称搜索、部署状态筛选、注册时间范围。

**交互逻辑**：
- **注册即客户**：C端用户手机号注册登录成功后，后台客户列表自动新增一条客户记录。
- **客户详情**：点击进入 B02-A-a，分 Tab 展示客户全部信息（基本信息/小程序/推广位/部署进度）。

**特殊约束**：手机号脱敏展示（中间4位 `****`），客户记录不可手动删除，仅可停用。

---

#### B03 推广位管理（平台视角）

**列表页字段**：推广位ID、所属客户（公司名称/手机号）、推广位名称、绑定活动数、取链总次数、创建时间、状态（启用/停用）、操作（查看详情/配置平台参数）。

**默认排序**：创建时间倒序。

**筛选条件**：客户手机号/名称搜索、推广位ID搜索、状态筛选。

**交互逻辑**：
- 运营不直接创建/删除推广位，推广位由商户端自行管理。
- 运营可查看所有商户推广位、配置平台级参数（渠道标识、跟踪参数等，商户端不可见）。
- 推广位详情页展示该推广位所有取链记录（时间+活动+链接）。

**业务规则**：每个客户至少保留1个推广位（系统预置）；停用推广位后已生成链接不失效但不可再取新链。

---

#### B04 活动管理

**列表页字段**：活动名称、活动类型、供应商、开始时间、结束时间、排序值、上下架状态、取链次数、状态（未开始/进行中/已结束）、操作（编辑/查看数据/上架/下架）。

**默认排序**：排序值升序 → 开始时间倒序。

**筛选条件**：活动名称搜索、供应商筛选、状态筛选、时间范围。

**新增/编辑表单（B04-A-a）**：

| 字段 | 类型 | 校验规则 |
|------|------|----------|
| 活动名称 | 文本（max 30字） | 必填，不重复 |
| 活动类型 | 下拉单选 | 必填（点餐/优惠券/新人福利/自定义） |
| 供应商 | 下拉单选 | 必填，决定取链接口 |
| 活动时间 | 日期时间范围 | 必填，开始 < 结束 |
| 佣金比例/金额 | 数字 | 选填 |
| 第三方活动ID | 文本 | 选填，供应商侧的活动标识 |
| 链接模板 | 文本（含变量占位符） | 必填，如 `https://xxx.com?slot={slot_id}&act={act_id}` |
| 适用推广位 | 多选/全部 | 必填，选择哪些推广位可参与此活动 |
| 排序值 | 数字 | 必填，越小越靠前 |
| 活动图片 | 图片上传 | 选填 |
| 上架/下架 | 开关 | 默认下架 |

**交互逻辑**：
- 活动开始后自动变为"进行中"，到期自动变为"已结束"。
- 运营下架活动时，已取链接提示失效但保留记录。
- 支持查看每个活动的取链次数和点击数据（B04-B）。

---

#### B05 订阅推送管理

**列表页字段（推送任务）**：任务名称、推送模板、推送时间（固定11:30）、目标人群、状态（运行中/暂停/已结束）、最近推送人数、创建时间、操作（编辑/暂停/查看记录）。

**交互逻辑**：
- 推送时间固定为每日11:30。
- 推送内容引导用户点击进入C端小程序权益页。
- 推送执行记录（B05-C）：每日推送人数、成功数、失败数、点击数、点击率。

**推送模板（B05-A）**：

| 字段 | 类型 | 校验规则 |
|------|------|----------|
| 模板名称 | 文本（max 20字） | 必填 |
| 推送标题 | 文本（max 30字） | 必填，如"今日点餐福利已送达" |
| 推送摘要 | 文本域（max 80字） | 必填 |
| 跳转目标 | 下拉 | 固定"权益页" |
| 封面图片 | 图片上传 | 选填 |

---

#### B06 小程序部署管理（核心模块）

**列表页字段**：客户名称、小程序名称、授权状态（待授权/已授权）、名称审核状态（待提交/审核中/已通过/驳回）、备案状态（待提交/审核中/已通过）、部署状态（待部署/部署中/已上线/失败）、预计完成时间、操作（查看详情/推进节点）。

**部署流程（约7天）**：

```
授权 → 提交小程序名称 → 备案 → 部署上线
  ↓         ↓              ↓        ↓
扫码授权   审核名称      提交备案   完成部署
(第1天)   (第1-2天)    (第3-5天)  (第5-7天)
```

**部署详情页（B06-A-a）**：
- 时间线展示各节点进度（授权时间、名称提交时间、备案提交时间、部署完成时间）。
- 每个节点支持状态标记：待处理 → 处理中 → 已完成 → 异常。
- 支持运营手动推进节点、备注异常原因。

**授权记录（B06-B）**：商户扫码授权记录，含授权时间、授权账号、授权权限范围。

**备案管理（B06-C）**：各商户备案进度跟踪，支持按备案状态筛选。

---

#### B07 权益配置管理（全局）

**全局配置（B07-A）**：权益页默认展示模板、自动取链规则（单例配置，无列表页）。

**默认取链规则**：
- 每个商户注册后，系统自动为当前所有"进行中"且"已上架"的活动生成推广链接，关联该商户的默认推广位。
- 新活动上架时，自动为所有已部署商户的默认推广位取链。

**商户权益查阅（B07-B）**：运营可按商户查看其权益页当前实际展示内容，包括已关联的活动列表、取链记录、当前有效链接。只读查阅，不做编辑。

---

#### B08 小程序管理（平台视角）

**列表页字段（B08-A）**：小程序名称、AppID、所属客户（公司名称/手机号）、小程序类型、注册时间、订阅开关状态（已开启/已关闭）、状态（正常/审核中/已停用）、操作（查看详情/停用/恢复）。

**默认排序**：注册时间倒序。

**筛选条件**：小程序名称搜索、AppID搜索、所属客户搜索、状态筛选、订阅开关状态筛选。

**交互逻辑**：
- **商户端注册即入驻**：商户在客户管理后台提交小程序信息后，总后台列表自动新增一条记录。
- **小程序详情（B08-A-a）**：点击进入详情页，展示基础信息、订阅状态、运营数据。
- **订阅开关记录（B08-B）**：记录每个小程序订阅开关的操作日志（操作人/时间/类型/变更前后状态），只读。

**特殊约束**：小程序记录不可手动删除，仅可停用；AppID全局唯一。

---

#### B09 日报推送管理（平台视角）

**列表页字段（B09-A）**：推送批次号（P{YYYYMMDD}）、统计日期、推送客户数、成功数、失败数（>0标红）、推送时间、操作（查看详情）。

**默认排序**：推送时间倒序。

**筛选条件**：推送日期范围、推送状态（全部/全部成功/部分失败/全部失败）。

**推送详情页（B09-A-a）**：
- **推送概览**：批次号、统计日期、推送时间、推送客户总数、成功数、失败数。
- **客户送达明细**：客户姓名、脱敏手机号、有数据的推广位个数、当日汇总收入、当日汇总订单数、送达状态（成功/失败）、失败原因。

**交互逻辑**：
- **推送触发在外部**：运营在统计后台发起日报推送，SaaS通过接口接收推送命令和数据，后台仅展示推送记录。
- **客户匹配**：SaaS接收日报数据后，根据客户标识（手机号/编码）匹配SaaS系统中的商户账号。
- **失败处理**：匹配失败或分发失败的记录展示失败原因（未匹配到客户/客户已停用/推送通道异常），运营可查看。

**特殊约束**：推送记录只读，不可编辑或删除；日报数据由外部统计后台计算，SaaS仅负责接收与分发。
