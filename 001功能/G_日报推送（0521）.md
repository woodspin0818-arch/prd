# 日报推送：功能模块规格

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 日报推送 |
| 功能入口 | 商户端：小程序 → 我的 → 日报；后台：左侧菜单 → 日报推送管理 |
| 覆盖功能 | TX-X003（商户端日报查看）、TX-X004（后台日报推送管理） |
| 覆盖页面 | 商户端：日报列表页、日报详情页；后台：B09-A 日报推送列表、B09-A-a 推送详情页 |
| 时间 | 2026-05-21 |
| 依赖模块 | 推广位管理模块（日报数据按推广位维度汇总）、客户管理模块（推送时查询客户信息）、统计后台（外部系统，负责推广数据计算与汇总） |

> **系统架构说明**：日报的源数据由外部「统计后台」计算并汇总。SaaS 系统的职责是：接收统计后台的推送命令 → 查询客户信息（推广位等由统计后台预配置）→ 将日报数据推送至对应客户的商户端。SaaS 不负责数据计算，仅负责数据接收、匹配与分发。

## 功能 TX-X003：商户端日报查看

---

### 1. 需求背景与目标 (Why)

合作方客户需要每日查看其推广位的推广效果数据（收入、订单数），以便及时了解投放效果、调整推广策略。目前日报仅展示日期+收入+订单数，后续可按需扩展指标维度。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 操作路径图 (User Flow)

```
  进入日报列表        查看日报列表          点击某条日报         查看日报详情
  (我的→日报)   ───►  (日报列表页)   ───►  (点击日期行)  ───►  (日报详情)
       │                    │                    │                  │
       │                    │ 下拉刷新           │                  │ 返回
       ▼                    ▼                    ▼                  ▼
   (需登录)             刷新最新日报          进入详情            返回列表
```

| 步骤 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|----------|----------|----------|----------|
| 1 | 进入日报 | 日报列表页 | 已登录商户账号 | - |
| 2 | 查看日报列表 | 日报列表页 | 已有推送的日报数据 | - |
| 3 | 点击某条日报 | 日报列表页 | 该日期有日报数据 | 日报详情页 |
| 4 | 下拉刷新 | 日报列表页 | - | 列表刷新 |
| 5 | 返回 | 日报详情页 | - | 日报列表页 |

#### 2.2 状态流转图

```

                                           │
```

| 有日报数据 | 商户已收到至少 1 条日报 | 运营推送后 SaaS 分发成功 | 日报详情 |

#### 3.5 业务规则

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|----------|----------|----------|----------|
| RULE-RPT-01 | 日报数据由统计后台计算 | 运营触发推送 | SaaS 不计算日报数据，仅接收统计后台推送的结果并分发 | 统计后台推送失败时不生成日报 |
| RULE-RPT-02 | 日报按商户维度分发 | SaaS 收到统计后台推送 | SaaS 根据统计后台预配置的客户-推广位映射关系，将日报数据匹配到对应商户账号 | 匹配失败时记录异常日志，人工介入 |
| RULE-RPT-03 | 日报列表按日期倒序 | 列表查询 | 最新日期排在最前 | - |
| RULE-RPT-04 | 无数据日期不展示 | 列表查询 | 仅展示有日报数据的日期；未推送的日期不出现在列表中 | - |

## 功能 TX-X004：后台日报推送管理

---

### 1. 需求背景与目标 (Why)

运营人员需要在后台查看日报推送的状态和记录，确认每次推送是否成功触达客户，并在推送异常时能够定位问题。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 操作路径图 (User Flow)

```
  进入日报推送管理      查看推送记录列表       点击某条推送
  (后台→日报推送)  ───►  (B09-A列表)   ───►  (B09-A-a详情)
       │                    │                    │
       │                    │                    ├─ 查看推送概览
       │                    │                    ├─ 查看推送明细（各客户送达状态）
       │                    │                    └─ 查看失败原因
       │                    │                    │
       ▼                    ▼                    ▼
   (需管理员登录)        列表展示             返回列表
```

| 步骤 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|----------|----------|----------|----------|
| 1 | 进入日报推送管理菜单 | B09-A 推送列表 | 管理员角色登录 | - |
| 2 | 筛选/搜索推送记录 | B09-A 列表 | - | 列表刷新 |
| 3 | 点击[详情] | B09-A 列表 | 推送记录存在 | B09-A-a 详情页 |

> **注意**：运营触发推送的操作在外部「统计后台 / 数据系统」完成，不在 SaaS 后台操作。SaaS 后台仅展示推送记录和送达状态。

## 模块汇总

> **核心说明**：
> 1. **SaaS 不计算数据**：日报的源数据（收入、订单数）由外部统计后台按推广位维度计算并汇总，SaaS 仅负责接收、匹配客户、分发至商户端。SaaS 不持有计算逻辑。
> 2. **推送触发在外部**：运营在统计后台/数据系统点击"推送"触发流程，SaaS 通过接口/回调接收命令和数据。SaaS 后台仅展示推送记录和送达状态，不提供"发起推送"按钮。
> 3. **客户匹配是关键**：统计后台预配置了客户的推广位等信息，SaaS 通过客户标识（手机号/编码）匹配到 SaaS 系统中的商户账号。匹配失败是主要的异常场景。
> 4. **当前日报仅两维度**：日期 + 收入 + 订单数。后续可按需扩展更多指标（如点击数、转化率等）。
> 5. **日报详情含推广位明细**：商户端日报详情页展示当日汇总 + 各推广位明细，方便客户按推广位分析效果。
