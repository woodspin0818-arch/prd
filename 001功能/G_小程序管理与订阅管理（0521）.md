# 小程序管理、入驻与订阅管理：功能模块规格

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 小程序管理、入驻与订阅管理 |
| 功能入口 | 商户端：我的 → 小程序管理（轻量注册）/ 左侧导航栏 → 极速开店（极速入驻）；后台：左侧菜单 → 小程序管理 |
| 覆盖功能 | TX-X007（商户端小程序管理与入驻）、TX-X008（订阅开关管理） |
| 覆盖页面 | 商户端：小程序列表页、轻量注册/编辑表单、极速入驻向导（P01）、入驻进度看板（P02）、小程序详情页；后台：B09-A 小程序列表、B09-A-a 小程序详情、B09-B 订阅开关记录 |
| 时间 | 2026-05-21 |
| 依赖模块 | 客户管理模块（商户登录态校验）、微信开放平台（AppID 校验）、微信第三方平台配置中心（极速入驻授权与部署）、OCR 服务（营业执照识别）、订阅推送管理模块（订阅后推送消息依赖 B05 的推送能力） |

> **系统预置规则**：
> 1. 商户注册登录后，小程序列表初始为空。商户可选择两条路径入驻：**轻量注册**（适用于已有微信小程序，仅需填写 AppID+名称+类型）或 **极速入驻**（适用于无小程序商户，OCR 识别→对公打款→扫码授权→自动部署上线）。
> 2. 订阅开关默认开启——即商户注册小程序后，C端消费者点击该小程序广告位时默认弹出微信订阅消息授权弹窗。
> 3. 极速入驻为单向线性流程，不支持回退操作，通过 P02 进度看板缓解商家等待焦虑。

## 功能 TX-X007：商户端小程序管理与入驻

---

### 1. 需求背景与目标 (Why)

商户客户需要将小程序入驻到平台，以便后续使用推广位取链、订阅推送等营销能力。针对两类商户场景：
- **已有微信小程序**的商户：通过轻量注册，填写 AppID + 名称 + 类型即可快速完成入驻。
- **尚无小程序**的商户：通过极速入驻向导，OCR 识别营业执照 → 自动填充 → 对公打款认证 → 扫码授权 → 自动备案部署上线，打造"传图-打款-扫码"三步傻瓜式体验。

---

#### 2.1 操作路径图 (User Flow)

**轻量注册路径**（适用于已有小程序商户）：

```
  (我的→小程序管理)  ───►  (列表页)  ───►  (轻量注册表单)  ───►  (保存)
       │                    │                    │                  │
       │                    │ 点击[编辑]         │ 校验失败         │ 保存成功
       ▼                    ▼                    ▼                  ▼
   (需登录)            编辑小程序表单         标红提示            返回列表
                            │                    │                  │
                            │ 点击[查看详情]     │ 保存             │ 取消
                            ▼                    ▼                  ▼
                        小程序详情页           返回列表            返回列表
                            │
                            │ 订阅开关操作
                            ▼
                        (见 TX-X008)
```

**极速入驻路径**（适用于尚无小程序商户）：

```
  进入极速入驻向导      上传营业执照+法人身份证     OCR自动填充+名称校验     点击提交
  (左侧导航→极速开店)  ───►  (P01 向导页)  ───►  (核对信息)  ───►  (发起快注)
       │                                                                  │
       │                                                                  ▼
       │                                                           进入 P02 进度看板
       │                                                                  │
       │   ┌──────────────────────────────────────────────────────────────┘
       ▼   ▼
  (P02 看板) ──► 复制打款信息 → 商家网银打款 → 微信认证通过回调
       │              │
       │              ▼
       │         管理员扫码授权 → 自动提审 + ICP备案 → 代码自动部署上线
       │
       └──► 每15秒轮询进度 / 微信回调自动推进状态
```

## 功能 TX-X008：订阅开关管理

---

### 1. 需求背景与目标 (Why)

商户需要自主控制其小程序内是否向C端消费者弹出微信订阅消息授权弹窗。当开关开启时，消费者点击广告位会先弹出微信订阅消息授权弹窗，消费者订阅后可接收平台的推送消息（如每日点餐通知）。商户可根据运营需求灵活开启或关闭此功能。

> **注意**：极速入驻中的小程序在部署上线前不展示订阅开关，部署完成后默认开启。

---

### 2. 业务流程与逻辑 (How - 宏观)

## 模块汇总

> **核心说明**：
> 1. **双路径入驻体系**：模块提供轻量注册（适用于已有小程序商户）和极速入驻（适用于无小程序商户）两条路径。轻量注册仅需 AppID+名称+类型即可完成；极速入驻通过 OCR→打款→扫码三步实现一站式开店。
> 2. **极速入驻为单向线性流程**：不支持回退，通过 P02 进度看板实时展示节点状态，利用体验版机制缓解商家等待焦虑。
> 3. **订阅开关默认开启**：为保障推送触达率，新注册小程序默认开启订阅弹窗（极速入驻的小程序在部署完成后自动开启）。商户可根据需要关闭或批量操作。
> 4. **订阅不阻断广告跳转**：无论消费者允许或拒绝订阅，均不影响广告位跳转，避免因订阅弹窗降低广告转化。
> 5. **AppID 全局唯一**：同一微信小程序 AppID 不可被多个商户重复注册，保证平台数据唯一性。
> 6. **双端权限分离**：商户端负责小程序的注册/入驻和订阅开关操作；后台运营可查看所有小程序详情和入驻进度，支持手动推进异常入驻节点。
> 7. **与 B05 推送管理的协作**：订阅开关开启 + 消费者已订阅 → 该消费者成为 B05 推送任务的目标人群，每日 11:30 可接收点餐通知推送。
> 8. **入驻中状态隔离**：极速入驻未完成的小程序不展示订阅开关，不支持编辑，不支持批量订阅操作，与已完成入驻的小程序在交互上有明确区分。

3. **订阅弹窗触发频率**：同一消费者每次点击广告位都弹窗，还是仅首次？若消费者曾拒绝订阅，多久后重新弹窗？当前策略：已订阅消费者不弹窗，未订阅消费者每次点击弹窗（受微信侧频率控制）。
4. **批量操作范围**：当前设计为对"当前商户全部已完成入驻的小程序"批量操作。是否改为支持"勾选多个小程序"的灵活批量操作？

6. **小程序类型枚举**：当前预设"点餐/外卖/电商/生活服务/其他"，是否需要扩展或调整？

9. **入驻节点可调整性**：极速入驻的五个节点（资料建立→打款认证→扫码授权→ICP备案→代码部署）是否需要支持跳过/合并？
10. **入驻超时处理**：若商家在"待打款认证"阶段长期未打款（如超过30天），是否需要自动取消或提醒？
