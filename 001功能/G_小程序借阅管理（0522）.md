# 小程序借阅管理：功能模块规格

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 小程序借阅管理 |
| 功能入口 | 商户端：极速入驻 P02 进度看板（"申请借阅小程序"入口）；后台：左侧菜单 → 小程序管理 → 借阅管理 |
| 覆盖功能 | TX-X009（商户端借阅申请与管理）、TX-X010（后台借阅池管理与回收） |
| 覆盖页面 | 商户端：P02 借阅申请卡片、P02-B 借阅状态页、小程序列表中借阅标记；后台：B10-A 借阅池管理、B10-B 借阅记录列表、B10-B-a 借阅详情 |
| 时间 | 2026-05-22 |
| 依赖模块 | 小程序管理与入驻模块（TX-X007）、微信第三方平台配置中心、数据统计平台（DAU 数据） |

> **核心场景**：
> 1. 商户选择极速入驻后，进入"待打款认证"→"待扫码授权"→"部署上线中"等漫长等待期（通常 7-15 天）。在此期间商户无小程序可用，无法开展推广业务。
> 2. 平台提供"借阅小程序"服务：从借阅池中分配一个已就绪的小程序给商户临时使用，商户可立即开展推广位取链和 C 端投放。
> 3. 当商户自有小程序部署上线后，且借阅小程序的日活（DAU）降至 50 以下时，平台自动回收借阅小程序，商户切换至自有小程序。
> 4. 借阅期间借阅小程序的取链、推广位、订阅等能力与自有小程序一致。

## 功能 TX-X009：商户端小程序借阅申请与管理

---

### 1. 需求背景与目标 (Why)

商户在极速入驻的 7-15 天空档期内无小程序可用，无法开展推广业务获取收入，造成商户流失和客诉。本功能通过在入驻等待期向商户"借出"平台预备小程序，使商户可以立即开展推广业务，实现"边入驻边运营"的体验闭环。借阅小程序的回收条件为：①商户自有小程序已部署上线，且②借阅小程序 DAU < 50，确保回收时不影响商户的存量用户。

---

### 2. 借阅生命周期全景

#### 2.1 核心流程图

                                        │                                │
                                        │ 审核驳回                       │ 自有小程序上线

                                                                  ┌──────────────┐
                                                                  │  监测中       │
                                                                  │  (monitoring)│
                                                                  └──────┬───────┘
                                                                         │
                                                           DAU < 50 且    │
                                                           自有小程序已发布│

                                                                  │  待回收       │

                                                                         │

#### 2.2 借阅状态说明表

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 | C端行为 |
|----------|----------|----------|-------------|---------|
| 无借阅 | 商户未申请借阅，或借阅已回收 | 商户初始态 / 借阅回收后 | 待审核 | - |
| 待审核 | 商户已提交借阅申请，等待平台审核 | 商户在 P02 提交借阅申请 | 借阅中、无借阅（驳回） | - |
| 借阅中 | 借阅小程序已分配给商户，商户可正常使用 | 平台审核通过并分配小程序 | 监测中 | 借阅小程序正常服务 |
| 监测中 | 商户自有小程序已上线，正在监测借阅小程序 DAU | 商户自有小程序状态变更为"已发布" | 待回收、借阅中（DAU 反弹回 ≥50） | 借阅小程序仍可用，启动倒计时提醒 |
| 待回收 | 借阅小程序 DAU < 50，等待执行回收 | 监测中 + DAU < 50 + 自有小程序已发布 | 已回收 | 借阅小程序仍可用（缓冲期），展示切换提醒 |
| 已回收 | 借阅小程序已回收至借阅池 | 平台执行回收操作 | -（终态） | 跳转至自有小程序 |

#### 2.3 完整生命周期时间线

```
  时间轴 →

  T0: 商户提交极速入驻资料（P01）
  T1: 商户申请借阅小程序（P02 看板中触发）
  T2: 平台审核通过，分配借阅小程序（通常 T0+1 工作日内）
  T3~Tn: 商户使用借阅小程序正常运营（推广位取链、C端投放、订阅推送）
  ...
  Tx: 商户自有小程序部署上线（状态 → 已发布）
  Tx+1: 系统进入监测期，每日检查借阅小程序 DAU
  Ty: 借阅小程序 DAU < 50（连续 N 天，见业务规则）
  Ty+1: 系统通知商户即将回收，启动缓冲期
  Ty+N: 缓冲期结束，执行回收，借阅小程序回到借阅池
  Tz: 商户 C 端流量完全切换至自有小程序
```

#### 3.4 借阅小程序切换——C端流量切换逻辑

当借阅小程序回收后，C 端消费者需无缝切换至商户自有小程序。

##### 3.4.1 切换机制

```
C端消费者扫码/点击广告位
       │
       ▼
  ┌─────────────────────┐
  │ 判断：借阅小程序      │
  │ 是否已被回收？       │
  ├─────────────────────┤
  │ 否 → 正常打开借阅    │
  │      小程序          │
  │                     │
  │ 是 → 读取绑定关系    │
  │      └→ 获取商户自有 │
  │         小程序AppID  │
  │      └→ 302跳转至   │
  │         自有小程序   │
  └─────────────────────┘
```

##### 3.4.2 切换要点

- 广告位取链时，后端根据 `miniapp_id` 判断是否为借阅小程序
- 若借阅小程序已回收（status = recycled），广告位链接自动替换为商户自有小程序路径
- 推广位、权益页配置在回收时自动迁移至自有小程序（见 5.3 定时任务）

## 功能 TX-X010：后台借阅池管理与回收

---

### 1. 需求背景与目标 (Why)

平台运营需要一个借阅小程序池来支撑借阅业务。运营人员可以管理借阅池中的小程序（录入/下架/查看状态），审核商户的借阅申请，监控借阅小程序的 DAU 数据，以及处理回收流程。自动回收调度器确保借阅小程序在满足条件后及时回收。

## 6. 业务规则

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|----------|----------|----------|----------|
| RULE-BRW-01 | 借阅仅对极速入驻商户开放 | 商户申请借阅 | 校验商户入驻方式为"极速入驻"且入驻状态 ≥ 待打款认证 | 轻量注册商户不展示借阅入口 |
| RULE-BRW-02 | 一商户同时仅可借阅一个小程序 | 商户申请借阅 / 后台审核 | 校验商户无进行中的借阅记录（status in [pending, lent, monitoring, reclaiming]） | 返回错误码 700001 |
| RULE-BRW-03 | 借阅池小程序不可重复借出 | 后台审核分配 | 校验 pool.status = available | 若小程序已被借出，提示"该小程序已被借出" |
| RULE-BRW-04 | 回收双条件 | DAU 监控定时任务 | 自有小程序 status = published（已发布）AND 借阅小程序 DAU < 50（连续 3 天） | 仅满足单条件时保持 monitoring 状态 |
| RULE-BRW-05 | DAU < 50 连续 3 天确认 | CRON-BRW-02 每日检查 | 连续 3 天 DAU < 50 才触发回收，避免单日波动 | 某天 DAU ≥ 50 则重置计数器 |
| RULE-BRW-06 | 回收缓冲期 3 天 | 触发回收后 | 状态变为 reclaiming，给商户 3 天切换时间，期间借阅小程序仍可用 | 缓冲期到后自动执行回收 |
| RULE-BRW-07 | 回收时数据迁移 | 执行回收操作 | ①推广位关联的 miniapp_id 替换为自有小程序ID ②权益页配置拷贝至自有小程序 ③通知商户回收完成 | 迁移失败告警并重试 |
| RULE-BRW-08 | 借阅小程序订阅数据不迁移 | 执行回收操作 | 借阅期间的订阅消费者数据随借阅小程序回池，不与自有小程序合并 | 避免数据混淆 |
| RULE-BRW-09 | 借阅池小程序类型优先匹配 | 后台审核分配 | 系统推荐与商户入驻选择类型一致的借阅小程序，若无同类型则推荐"其他"类型 | 运营可手动选择其他类型 |
| RULE-BRW-10 | 借阅小程序回收后回池 | 回收完成 | pool.status 从 lent 改为 available，total_lend_count +1，current_merchant_id 清空 | 若小程序需维护则改为 maintenance |

## 模块汇总

> **核心说明**：
> 1. **借阅定位**：借阅是极速入驻的配套增值服务，在商户 7-15 天空档期内提供可运营的小程序，让商户实现"边入驻边运营"。
> 2. **借阅与自有双轨制**：商户同时存在借阅小程序和自有小程序（入驻中），两者在推广位、权益页、订阅等能力上独立运作，回收时进行数据迁移。
> 3. **回收双条件加缓冲**：自有小程序上线 + 借阅小程序 DAU < 50（连续 3 天）→ 3 天缓冲期 → 自动回收。这个设计确保回收时商户已有可用的自有小程序，且大部分 C 端流量已自然迁移至自有小程序。
> 4. **DAU 阈值 50**：低于 50 日活说明该借阅小程序的 C 端用户已大部分流失或已迁移至自有小程序，此时回收对商户业务影响最小。
> 5. **借阅池运营**：平台需要持续维护一个足够大的借阅池（建议 ≥ 在借数量 × 2），并定期轮换/维护借阅小程序。
> 6. **与极速入驻强绑定**：借阅能力仅对极速入驻商户开放，轻量注册商户已有自己的小程序，不需要借阅。
> 7. **订阅数据隔离**：借阅期间的订阅消费者数据不迁移至自有小程序，鼓励商户在自有小程序重新运营订阅。
