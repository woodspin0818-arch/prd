# 活动管理：功能模块规格

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 活动管理 |
| 功能入口 | 商户端：小程序 → 权益页 → 活动列表 → 取链；后台：左侧菜单 → 活动管理 |
| 覆盖功能 | TX-X005（后台活动管理）、TX-X006（商户端活动列表与取链） |
| 覆盖页面 | 商户端：活动列表页、取链结果页；后台：B04-A 活动列表、B04-A-a 新增/编辑活动表单 |
| 时间 | 2026-05-21 |
| 依赖模块 | 推广位管理模块（取链时需选择推广位，推广位预配置了各平台参数）、客户管理模块（客户登录态校验） |

> **系统说明**：运营在后台配置活动信息（含第三方活动ID），配置完成后商户端即可在活动列表看到已上架的活动。商户选择活动后，系统根据活动的供应商（美团/饿了么）调用对应平台的取链接口，结合推广位的平台参数生成推广链接。**第三方活动ID** 是本模块的核心字段，用于锁定第三方平台对应的活动，取链时必须传入。

## 功能 TX-X005：后台活动管理

---

### 1. 需求背景与目标 (Why)

运营人员需要在后台统一配置和管理营销活动，作为商户端取链的数据源。通过后台配置活动的基础信息、供应商、佣金、第三方活动ID 等字段，控制商户端可见的活动范围和展示顺序。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 操作路径图 (User Flow)

```
  进入活动列表        筛选/搜索活动        点击[新增活动]       填写活动信息
  (后台→活动管理)  ───►  (B04-A列表)  ───►  (B04-A-a表单)  ───►  (保存)
       │                    │                    │                  │
       │                    │ 点击[编辑]         │ 校验失败         │ 保存成功
       ▼                    ▼                    ▼                  ▼
   (需管理员登录)      编辑活动表单           标红提示            返回列表
                            │
                            │ 点击[上架/下架]
                            ▼
                        状态切换（即时生效）
```

| 步骤 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|----------|----------|----------|----------|
| 1 | 进入活动管理菜单 | B04-A 活动列表 | 管理员角色登录 | - |
| 2 | 筛选/搜索活动 | B04-A 列表 | - | 列表刷新 |
| 3 | 点击[新增活动] | B04-A 列表 | - | B04-A-a 新增表单 |
| 4 | 填写活动信息并保存 | B04-A-a 表单 | 校验通过 | 返回列表，新增一行 |
| 5 | 点击[编辑] | B04-A 列表 | 活动存在 | B04-A-a 编辑表单（回填数据） |
| 6 | 修改信息并保存 | B04-A-a 表单 | 校验通过 | 返回列表，刷新该行 |
| 7 | 点击[上架]/[下架] | B04-A 列表 | 活动存在 | 即时切换状态，商户端同步生效/失效 |

#### 2.2 状态流转图

```
  ┌──────────────┐   运营[新增活动]     ┌──────────────┐
  │   无活动      │──────────────────►│   已下架      │
  │   (初始态)   │   默认下架         │   (下架态)   │
  └──────────────┘                    └──────────────┘
                                           │
                                           │ 运营[上架]
                                           ▼
                                      ┌──────────────┐
                                      │   已上架      │
                                      │   (上架态)   │
                                      └──────────────┘
                                           │
                                           │ 运营[下架]
                                           ▼
                                      ┌──────────────┐
                                      │   已下架      │
                                      │   (下架态)   │
                                      └──────────────┘
```

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 |
|----------|----------|----------|-------------|
| 已下架 | 活动不可在商户端展示 | 新增活动默认下架 / 运营手动下架 | 已上架 |
| 已上架 | 活动在商户端可见，商户可取链 | 运营点击[上架] | 已下架 |

> **说明**：活动状态仅控制商户端可见性。已上架的活动下架后，商户端不再展示该活动，但已生成的推广链接不失效。

##### 接口详情

---

**API-ACT-01：[获取活动列表]**

**调用场景**：

| 项目 | 说明 |

## 功能 TX-X006：商户端活动列表与取链

---

### 1. 需求背景与目标 (Why)

商户客户在权益页需要查看当前可参与的营销活动列表，并选择活动生成推广链接进行投放。商户端仅展示已上架的活动，按后台配置的排序值排列。商户选择活动后，系统根据活动的供应商和第三方活动ID，结合商户推广位的平台参数，调用第三方取链接口生成推广链接。

#### 3.5 业务规则

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 | 异常处理 |

## 模块汇总

> **核心说明**：
> 1. **第三方活动ID 是核心纽带**：每个活动有唯一的第三方活动ID（如美团活动ID、饿了么活动ID），取链时系统将此 ID 连同推广位的平台参数（APPKEY/SECRET等）传给第三方接口，生成带跟踪参数的推广链接。
> 2. **状态控制可见性**：活动仅有两个状态（上架/下架），商户端仅可见已上架的活动。下架后已生成的推广链接不失效。
> 3. **展示字段与逻辑字段分离**：佣金、有效期、会场推广规则均为展示用途，不影响实际取链逻辑。实际取链仅依赖：供应商 + 第三方活动ID + 推广位平台参数。
> 4. **排序控制客户端展示**：运营通过排序值控制商户端活动卡片的展示顺序，数字越小越靠前。
> 5. **推广位是取链的前置条件**：商户取链时必须选择推广位，推广位的平台参数（APPKEY/SECRET等）由运营在推广位详情页预配置。
