---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 推广位管理 |
| 功能入口 | 商户端：小程序 → 我的 → 推广位管理；后台：左侧菜单 → 推广位管理 |

| 覆盖页面 | 商户端：推广位列表页、新增/编辑推广位表单；后台：B03-A 推广位列表、B03-A-a 推广位详情/配置页 |

> **系统预置规则**：每个客户注册成功后，系统自动创建 1 个默认推广位，推广位名称为"默认推广位"，推广位ID 按规则生成（见 RULE-001）。默认推广位不可删除。

## 功能 TX-X001：商户端推广位管理

---

### 1. 需求背景与目标 (Why)

商户客户需要管理自己的推广位，用于后续取链投放。每个商户注册后自动获得一个默认推广位，商户可自行新增更多推广位、编辑推广位名称、设置默认推广位、删除非默认推广位。取链时系统自动选中默认推广位，简化操作流程。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 操作路径图 (User Flow)

```
  进入推广位列表      查看推广位列表        点击[新增推广位]      填写名称
  (我的→推广位管理)  ───►  (列表页)  ───►  (新增表单)  ───►  (输入名称)
       │                    │                    │                  │
       │                    │ 点击[设为默认]     │ 点击[编辑]       │ 保存
       ▼                    ▼                    ▼                  ▼
   (需登录)            切换默认推广位         编辑表单            返回列表
                            │                    │                  │
                            │ 点击[删除]         │ 保存             │ 取消
                            ▼                    ▼                  ▼
                        二次确认删除           返回列表            返回列表
                       (默认推广位不可删)
```

| 步骤 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|----------|----------|----------|----------|
| 1 | 进入推广位管理 | 推广位列表页 | 已登录商户账号 | - |
| 2 | 查看推广位列表 | 推广位列表页 | 至少已有1个默认推广位 | - |
| 3 | 点击[新增推广位] | 推广位列表页 | - | 新增推广位表单 |
| 4 | 填写名称并保存 | 新增表单 | 名称不为空 | 返回列表页，新增一行 |
| 5 | 点击[编辑] | 推广位列表页 | - | 编辑推广位表单（回填名称） |
| 6 | 修改名称并保存 | 编辑表单 | 名称不为空 | 返回列表页，更新名称 |
| 7 | 点击[设为默认] | 推广位列表页 | 该推广位非当前默认 | 切换默认推广位，原默认取消 |
| 8 | 点击[删除] | 推广位列表页 | 该推广位非默认推广位 | 二次确认 → 删除 → 列表刷新 |
| 9 | 取链时选择推广位 | 取链页面 | 已有至少1个推广位 | 默认选中默认推广位 |

  │   无推广位    │──────────────────►│   默认推广位   │
  │   (初始态)   │   系统自动创建      │   (1个)       │

                                           │ 商户[新增推广位]

                                      ┌──────────────┐
                                      │  多个推广位   │
                                      │  (≥2个)      │
                                      └──────────────┘
                                           │
                           ┌───────────────┼───────────────┐
                           │               │               │

                           │               │               │
                           ▼               ▼               ▼
                      默认推广位切换    名称更新        推广位减少

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 可触达的状态 |

| 规则编号 | 规则描述 | 触发条件 | 处理逻辑 | 异常处理 |
|----------|----------|----------|----------|----------|
| RULE-001 | 推广位ID 生成规则 | 新增推广位时 | ID 格式：xxxxx-{客户ID}-{序号}。其中客户ID为客户的客户ID，序号为该客户推广位创建顺序（从1开始递增，如第5个推广位则带5）。示例：xxxxx-14-5 表示客户ID=14的客户的第5个推广位 | - |
| RULE-002 | 注册自动创建默认推广位 | 商户注册成功 | 系统自动插入一条推广位记录：slot_name = "默认推广位"，is_default = true，slot_id 按 RULE-001 规则生成（序号为1） | 创建失败则回滚注册流程或异步补偿 |
| RULE-003 | 默认推广位不可删除 | 客户点击删除默认推广位 | 前端：默认推广位行不展示[删除]按钮；后端：API-SLOT-04 校验 is_default，若为 true 则拒绝删除 | 返回错误码 400001 |
| RULE-004 | 默认推广位唯一 | 客户点击[设为默认] | 后端 API-SLOT-03 在事务中将该商户所有推广位的 is_default 改为 false，再将目标推广位 is_default 改为 true | - |
| RULE-005 | 取链默认选中 | 进入取链页面时 | 取链页面的推广位选择器默认选中 is_default = true 的推广位；如商户修改了默认推广位，取链自动跟随新的默认推广位 | 若商户无默认推广位（极端情况），选中第一个推广位 |
| RULE-006 | 同一商户下推广位名称唯一 | 新增/编辑保存时 | 后端校验同商户下 slot_name 是否重复（不含已删除的） | 返回错误码 400002 |
| RULE-008 | 删除推广位时迁移链接 | 删除推广位时 | 若该推广位已被活动使用（有关联的推广链接），删除时将链接的推广位关联自动切换为默认推广位；若未被使用则直接删除 | 删除前弹窗告知客户迁移影响 |

## 模块汇总

> **核心说明**：
> 1. **双端管理分离**：商户端（TX-X001）聚焦推广位的 CRUD 和默认推广位管理；后台（TX-X002）聚焦推广位的平台参数配置和数据查看，运营不直接创建/删除推广位。
> 2. **推广位ID 规则统一**：格式 xxxxx-{客户ID}-{序号}，客户ID为客户的客户ID，序号为该客户推广位创建顺序（从1开始递增），全局唯一。示例：xxxxx-14-5 表示客户ID=14的客户的第5个推广位。
> 3. **默认推广位保护**：每个商户始终存在且仅有一个默认推广位（is_default = true），注册时自动创建，不可删除，取链时自动选中。
> 4. **平台参数独立配置**：每个推广位独立配置美团/饿了么/淘宝联盟的参数，运营后台可见可编辑，商户端不可见。
> 5. **删除迁移链接**：推广位删除时，若该推广位已被活动使用，系统自动将关联链接迁移至默认推广位，并在删除前弹窗告知客户。
