# 商户推广工作台（首页概览）：功能模块规格

---

## 模块概览

| 项目 | 内容 |
|------|------|
| 模块名称 | 商户推广工作台（首页概览 Dashboard） |
| 功能入口 | 商户端 Web 后台：登录后默认进入首页；左侧菜单 → 首页概览 |
| 覆盖功能 | TX-X009（商户推广工作台）、TX-X003（商户端日报查看-昨日数据摘要）、TX-X006（商户端活动取链-快捷入口） |
| 覆盖页面 | 商户端：C01 首页概览（商户仪表盘） |
| 时间 | 2026-05-21 |
| 依赖模块 | 日报推送模块（昨日数据区域数据来源）、活动管理模块（取链模块活动卡片数据来源）、推广位管理模块（取链时选择推广位）、财务管理模块（账户余额/提现数据来源） |

> **设计定位**：商户推广工作台是商户登录后的默认着陆页。它将「昨日收入数据」「账户资金概览」「活动取链」三大核心场景聚合在一个页面中，让商户无需跳转即可完成"看数据→管资金→推广活动"的日常工作闭环。页面采用上下分区布局：顶部为数据概览区（昨日收入 + 账户余额），底部为业务操作区（供应商Tab + 活动卡片网格）。

## 功能 TX-X009：商户推广工作台

---

### 1. 需求背景与目标 (Why)

商户登录后台后需要一个统一的"工作台"页面：一眼看到昨日推广收入和账户资金状况，同时能快速找到可用活动并生成推广链接。当前首页仅展示简单的指标卡片和快捷入口，缺乏可直接操作的活动取链能力和收入数据可视化。升级为"推广工作台"可将商户的核心工作流（看收入→选活动→生成链接）收敛到单一页面，降低操作路径长度。

---

### 2. 业务流程与逻辑 (How - 宏观)

#### 2.1 操作路径图 (User Flow)

```
  登录商户后台         查看工作台首页         浏览活动卡片          选择供应商Tab
  (登录)         ───►  (C01 首页概览)   ───►  (取链模块)     ───►  (切换Tab)
       │                    │                    │                    │
       │                    │                    │ 点击[立即推广]     │ 切换供应商
       ▼                    ▼                    ▼                    ▼
   (默认着陆页)        三大模块并行         生成推广链接          刷新活动卡片
                      ① 昨日数据
                      ② 财务数据
                      ③ 取链模块
                         │
                         ├─ 点击[立即推广] → 取链确认弹窗 → 生成推广链接 → 复制/分享
                         ├─ 点击[提现]     → 提现申请流程
                         └─ 点击日报区域    → 跳转日报列表页
```

| 步骤 | 触发动作 | 所在页面 | 前置条件 | 后续页面 |
|------|----------|----------|----------|----------|
| 1 | 登录商户后台 | 登录页 | 已注册商户账号 | C01 首页概览 |
| 2 | 查看昨日收入数据 | C01 首页 | 已有推送的日报数据 | 点击可跳转日报详情 |
| 3 | 查看账户余额 | C01 首页 | 已绑定提现账户 | 点击[提现]进入提现流程 |
| 4 | 切换供应商Tab | C01 首页 | 后台已配置对应供应商的活动 | 活动卡片网格刷新 |
| 5 | 点击[立即推广] | C01 首页 | 活动已上架 | 取链确认弹窗 |
| 6 | 生成推广链接 | 取链确认弹窗 | 已选活动+推广位 | 展示推广链接，可复制/分享 |

#### 2.2 页面状态流转图

```
  ┌──────────────────┐   运营配置活动+推送日报    ┌──────────────────┐
  │   空工作台        │──────────────────────►│   完整工作台       │
  │   (商户刚注册)    │                        │   (数据+活动齐全)  │
  └──────────────────┘                        └──────────────────┘
        │                                              │

  ┌──────────────────┐                        ┌──────────────────┐

  │   (有收入无活动,  │                        │   (取链模块空状态)│

```

**状态说明表**：

| 状态名称 | 状态含义 | 进入条件 | 页面表现 |
|----------|----------|----------|----------|
| 空工作台 | 商户刚注册，无任何数据 | 注册后首次登录，尚未推送日报，无上架活动 | 昨日数据显示"--"，取链模块展示空状态 |
| 部分数据态 | 仅有收入数据无活动，或仅有活动无收入数据 | 日报已推送但无上架活动 / 有上架活动但日报尚未推送 | 有数据区域正常展示，无数据区域显示占位 |
| 完整工作台 | 收入数据和活动均正常 | 日报已推送 + 至少1个上架活动 | 三大模块均正常展示 |
| 无可用活动 | 所有活动下架，取链模块不可用 | 运营将所有活动下架 | 取链模块展示空状态插画 |

#### 3.2 模块②：昨日数据（收入概览区）

##### 3.2.1 模块定位

位于页面顶部左侧，约占顶部区域 2/3 宽度。展示商户最关心的昨日推广收入核心指标，数据来源于日报推送系统。

##### 3.2.2 组件规格

**卡片容器**：
- 白色背景卡片，圆角 8px，轻微阴影
- 标题栏：「收入数据概览」，右侧弱化提示文字"只统计使用专属渠道的链接收入"（字号 12px，灰色 #999）

**核心指标区（横向排列）**：

| 指标 | 数据来源 | 展示格式 | 字号/字重 |
|------|----------|----------|-----------|
| 昨日预估佣金(元) | API-DASH-01 total_income | ¥1,250.00（两位小数） | 32px / Bold，主色 #333 |
| 昨日订单数 | API-DASH-01 total_orders | 32（整数） | 32px / Bold，主色 #333 |

**辅助信息**：
- 每个指标下方展示环比变化（如 "较前日 ↑12.5%"），需后端计算并返回
- 无前日数据时不展示环比

##### 3.2.3 组件状态表

| 组件编号 | 状态 | 视觉表现 | 触发条件 |
|----------|------|----------|----------|
| C-DASH-01 | 正常 | 指标数字正常展示，含环比变化 | 接口返回昨日数据成功 |
| C-DASH-01 | 加载中 | 骨架屏占位（两个数字区域灰色矩形占位） | 接口请求中 |
| C-DASH-01 | 无数据 | 两个指标均显示"--"，无环比 | 昨日无日报推送（如刚注册） |
| C-DASH-01 | 错误 | 指标区显示错误提示 + [重试] 按钮 | 接口异常/网络超时 |

##### 3.2.4 交互逻辑

| 交互编号 | 交互描述 | 触发元素 | 前置条件 | 处理逻辑 | 执行结果/跳转 |
|----------|----------|----------|----------|----------|-------------|
| INT-DASH-01 | 查看日报详情 | 点击「收入数据概览」卡片 | 有昨日数据 | 跳转至日报详情页（C05-A-a），传入昨日日期 | 日报详情页 |
| INT-DASH-02 | 页面加载获取数据 | 进入首页 / 登录后默认跳转 | 已登录 | 调 API-DASH-01 获取昨日收入摘要 | 昨日数据区域渲染 |
| INT-DASH-03 | 重试加载 | [重试] 按钮 | 接口异常 | 重新调 API-DASH-01 | 刷新昨日数据区域 |

位于页面下方，占据全屏宽度。这是商户日常推广操作的核心区域，采用"供应商Tab切换 + 活动卡片网格"的布局，让商户快速浏览并推广活动。

| 元素 | 说明 |
|------|------|

- 自适应网格布局，每行 3 张卡片（大屏），响应式可缩至 2 列

## 模块汇总

> **核心说明**：
> 1. **三合一工作台**：商户推广工作台将"数据监控（昨日收入）" + "资金管理（余额/提现）" + "业务操作（活动取链）"整合为单一页面，是商户日常运营的起点和终点。
> 2. **数据来源于已有模块**：昨日收入数据复用「日报推送」模块的底层数据（取最近一次推送的日报），活动卡片数据复用「活动管理」模块中已上架的活动列表。工作台本身不产生新数据，是已有数据的聚合展示层。
> 3. **取链流程收敛**：活动取链不再需要跳转到独立的权益页，商户在首页直接点击[立即推广]即可完成取链。复杂配置场景仍可通过导航进入完整权益页。
> 4. **提现为新增能力**：账户余额、提现申请等功能为本模块新增的财务能力，需后端搭建对应的资金账户体系和提现审核流程。最低提现金额100元，单笔审核制。
> 5. **并行加载提升体验**：三个模块的数据接口并行请求，任何单一接口失败不影响其他模块的正常渲染，保证首页整体可用性。
> 6. **活动主图字段需求**：当前活动管理模块的活动表单中缺少活动主图字段，为支撑首页卡片的主图展示，需在活动管理模块 B04-A-a 表单页新增「活动主图」上传字段。
